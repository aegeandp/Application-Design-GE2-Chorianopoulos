# Application-Design-GE2-Chorianopoulos
Ο σκοπός μας είναι να αναπτύξουμε μια  υπηρεσία (ας την ονομάσουμε "AquaSense"), η οποία θα αποτελεί μια καινοτόμα πλατφόρμα διάχυτου υπολογισμού που θα μπορεί να ενημερώνει σε **πραγματικό χρόνο** για την **αντιμετώπιση των επιπτώσεων της κλιματικής αλλαγής,** με έμφαση στη **λειψυδρία** και τα ακραία καιρικά φαινόμενα.

**Υποθέσεις/Πλαίσιο:**

* **Γεωγραφική Περιοχή:** Νησιωτική περιοχή της Ελλάδας (π.χ., Κυκλάδες) ή αγροτικές περιοχές με έντονα προβλήματα λειψυδρίας και ξηρασίας.

* **Χρήστες: Αγρότες/Κτηνοτρόφοι** (βασικός χρήστης) και **Καθημερινοί Κάτοικοι/Τουρίστες** (δευτερεύων χρήστης).

* **Τεχνολογικό Περιβάλλον:** Χρήση **κινητής εφαρμογής** (Mobile App), **φορετής συσκευής** (Wearable - π.χ., smartwatch) και **δικτυωμένων αισθητήρων** (Ambient Computing - π.χ., έξυπνοι μετρητές υγρασίας εδάφους/ποτίσματος, μετεωρολογικοί σταθμοί) για τη συλλογή **τοπικών και απομακρυσμένων δεδομένων.**

Η υπηρεσία θα παρέχει **επίγνωση πλαισίου (context-awareness)**, λαμβάνοντας δεδομένα από τους αισθητήρες (π.χ., υγρασία εδάφους, στάθμη δεξαμενών, τοπική πρόγνωση καιρού) και παρέχοντας **προσωποποιημένες, άμεσες οδηγίες δράσης** για τη διαχείριση του νερού.

Τα κριτήρια ποιότητας επικεντρώνονται στην αποτελεσματικότητα της διάδρασης για την έγκαιρη προστασία και τη βέλτιστη διαχείριση πόρων στο συγκεκριμένο πλαίσιο.

1. **Άμεση και Αξιόπιστη Ειδοποίηση Κινδύνου:** Η εφαρμογή πρέπει να επιτρέπει την παράδοση **κρίσιμων προειδοποιήσεων** (π.χ., επικείμενος καύσωνας, κίνδυνος πυρκαγιάς λόγω ξηρασίας, ξαφνική  πτώση στάθμης δεξαμενής) με **ελάχιστο λανθάνοντα χρόνο (low latency)** σε όλες τις συσκευές.

1. **Αποτελεσματικότητα Δράσης (Actionability):** Η διάδραση πρέπει να μετατρέπει τα δεδομένα σε άμεσες, σαφείς, και πραγματοποιήσιμες οδηγίες (π.χ., «Μειώστε το πότισμα κατά 30% αύριο», «Κλείστε την παροχή νερού στη ζώνη Β», «Προσοχή: Κίνδυνος θερμοπληξίας»), επιτρέποντας στον χρήστη να δράσει εντός δευτερολέπτων.

1. **Επίγνωση Πλαισίου και Εξατομίκευση (Context-Awareness & Personalization):** Η εφαρμογή πρέπει να αξιοποιεί τα τοπικά δεδομένα αισθητήρων για να παρέχει εξαιρετικά στοχευμένες πληροφορίες που να είναι συναφείς με τη συγκεκριμένη τοποθεσία και τον ρόλο του χρήστη (π.χ., διαφορετικές ειδοποιήσεις για αγρότη/κάτοικο).

1. **Συνέπεια Πληροφορίας σε Πολλαπλές Συσκευές (Cross-Device Consistency):** Η κρίσιμη πληροφορία πρέπει να είναι συνεκτική και εύκολα προσβάσιμη τόσο στον κινητό υπολογισμό (για λεπτομερή ανάλυση) όσο και στον φορετό υπολογισμό (για άμεση επίγνωση εν κινήσει).

Για το σκοπό της εργασία μας, μελετήσαμε 2 ερευνητικές εργασίες από τις οποίες η μία είναι τεχνική (IoT/Context) και η άλλη επικεντρωμένη στον χρήστη (Wearables/HCI). Επιλέξαμε αυτά τα δύο άρθρα γιατί θεωρούμε ότι μας παρέχουν ένα ισχυρό θεωρητικό πλαίσιο και λειτουργούν ως καθοδηγητικό νήμα για τη σχεδίαση και την αξιολόγηση της εφαρμογής μας.
